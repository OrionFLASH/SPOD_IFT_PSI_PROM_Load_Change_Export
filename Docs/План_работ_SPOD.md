# План работ по развитию SPOD-консолидации

## Фаза 1. Стабилизация текущей реализации

1. Проверка корректности объединения на реальных измененных данных между стендами.
2. Проверка, что для каждого стенда корректно применяются свои имена файлов из `config.json`.
3. Валидация корректности `source_stands` для всех 10 сущностей.
4. Проверка дедупликации:
   - повторный запуск без изменений;
   - запуск с частично новыми файлами.
5. Проверка сценария pre-check:
   - отсутствие одного/нескольких файлов;
   - отсутствие каталога стенда.
6. Проверка консольного отчета:
   - старт, найденные файлы, статус БД, статистика разночтений.

## Фаза 2. Расширение надежности

1. Добавить проверки схемы CSV по обязательным колонкам.
2. Добавить контроль кодировок и обработку битых строк.
3. Расширить unit/integration тесты.
4. Добавить тесты на правила подсветки строк и параметры форматирования из `config.json`.

## Фаза 3. Расширение отчетности

1. Добавить отдельный лист Excel:
   - `DIFF_REPORT` со сводкой расхождений.
2. Добавить экспорт технического отчета в JSON/CSV.
3. Добавить агрегаты по количеству совпадений/расхождений по стендам.
4. Добавить детализацию разночтений до уровня «какие поля отличаются» по одному ключу.

## Фаза 4. Эксплуатация

1. Подготовить регламент запуска и проверки.
2. Добавить версионирование схемы БД (миграции).
3. Добавить план архивирования старых запусков/логов.
4. Добавить versioned-шаблоны `config.json` для разных типов контуров/проектов.
