# Техническое задание (ТЗ): Консолидация настроек SPOD

## 1. Назначение

Разработать программу, которая:
- загружает CSV-настройки из стендов `IFT`, `PROM`, `PSI` из каталога `IN/SPOD`;
- объединяет однотипные данные между стендами;
- формирует Excel-отчет;
- сохраняет результаты и историю загрузок в SQLite;
- исключает повторную загрузку одинаковых файлов по `SHA-256`.

## 2. Входные данные

Источник: `IN/SPOD/{IFT|PROM|PSI}`.

Набор сущностей (10 типов):
- `CONTEST`
- `EMPLOYEE`
- `GROUP`
- `INDICATOR`
- `ORG_UNIT_V20`
- `REPORT`
- `REWARD`
- `REWARD-LINK`
- `SCHEDULE`
- `USER_ROLE`

## 3. Правила обработки

1. CSV читаются в исходном виде (разделитель `;`, без изменения структуры столбцов).
2. Для каждой строки формируются:
   - `business_key` по сущности;
   - `row_hash` по каноническому JSON строки.
3. Объединение выполняется по `(business_key, row_hash)`:
   - если строки эквивалентны во всех 3 стендах -> одна запись с `source_stands=PROM-IFT-PSI`;
   - если отличаются -> отдельные записи с маркировкой `PROM`/`IFT`/`PSI`.
4. В SQLite ведется история запусков и загрузок файлов.
5. Повторно не загружаются файлы, чьи hash уже есть в БД.

## 4. Выходные данные

1. Excel в `OUT/CSV`:
   - 1 лист на сущность;
   - лист `SUMMARY`.
2. SQLite в `OUT/DB`:
   - таблицы `runs`, `ingested_files`, `raw_rows`, `merged_rows`.
3. Логи в `log`:
   - `INFO_*`
   - `DEBUG_*`

## 5. Обязательные служебные поля в merged-данных

- `source_stands`
- `source_count`
- `is_equal_all`
- `diff_group_key`

## 6. Управление через конфиг

Параметры путей, сущностей, ключей, логирования, имен выходных файлов и runtime-режимов задаются в `config.json`.
