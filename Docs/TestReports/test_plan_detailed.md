# Подробная схема тестирования SPOD

## Цель

Проверить корректность работы пайплайна на:
- реальных данных из `IN/SPOD`;
- вариативных данных, полученных из реальных CSV с контролируемыми изменениями.

## Подход

1. Базовый прогон на исходном наборе.
2. Прогоны на копиях входных данных с отдельными модификациями.
3. Проверка регрессии по ключевым метрикам из консольного отчета.
4. Фиксация результатов в машинном (`.json`) и читаемом (`.md`) формате.

## Набор сценариев (статус на момент последнего прогона)

Сводка в `test_results_regression_latest.md`. Легенда: **[сделано]** — последний прогон PASS; **[не сделано]** — не запускался или FAIL.

| ID | Название | Статус последней проверки |
|----|----------|---------------------------|
| S0_BASELINE | Исходный `IN/SPOD`, 30 файлов, метрики строк/разночтений | **[сделано]** (PASS) |
| S0_QUOTE_PRESERVATION | Сохранность кавычек в `FILTER_PERIOD_ARR` (SCHEDULE/PROM) | **[сделано]** (PASS) |
| S1_PRECHECK_MISSING_FILE | Блокировка при отсутствии обязательного файла | **[сделано]** (PASS) |
| S2_NON_OPTIONAL_DIFFERENCE_DETECTED | Детект отличия по не-optional полю (GROUP) | **[сделано]** (PASS) |
| S3_OPTIONAL_ONE_STAND_IGNORED | Optional только на одном стенде — без ложных diff | **[сделано]** (PASS) |
| S4_OPTIONAL_TWO_STANDS_CONFLICT | Optional на двух стендах с разными значениями | **[сделано]** (PASS) |

### Краткое описание сценариев

1. **`S0_BASELINE`** — Вход: исходный `IN/SPOD`. Проверки: успешный запуск, чтение 30 файлов, расчёт строк/разночтений.

2. **`S0_QUOTE_PRESERVATION`** — Вход: исходный `IN/SPOD` (фрагмент SCHEDULE). Проверки: сохранность значения `FILTER_PERIOD_ARR` с кавычками.

3. **`S1_PRECHECK_MISSING_FILE`** — Вход: копия данных с удалённым обязательным файлом. Проверки: запуск блокируется, ожидаемая ошибка pre-check.

4. **`S2_NON_OPTIONAL_DIFFERENCE_DETECTED`** — Вход: копия, изменено не-optional поле. Проверки: рост показателя различий по сущности.

5. **`S3_OPTIONAL_ONE_STAND_IGNORED`** — Вход: копия, optional-колонка только на одном стенде (`EMPLOYEE`). Проверки: нет ложных различий.

6. **`S4_OPTIONAL_TWO_STANDS_CONFLICT`** — Вход: копия, optional в двух стендах с разными значениями. Проверки: различия фиксируются.

## Критерии успешности

- Все сценарии завершаются ожидаемым исходом.
- Метрики согласованы с логикой сценария.
- Результаты сохранены в:
  - `Docs/TestReports/test_results_regression_latest.md`
  - `Docs/TestReports/test_results_regression_latest.json`
