# ROADMAP: консолидация настроек SPOD

Документ объединяет прежние версии `ROADMAP.md` и `Docs/Roadmap_SPOD.md` без потери пунктов.

## Формат статусов для каждого пункта

| Маркер | Значение |
|--------|----------|
| `[v]` | Сделано |
| `[w]` | В работе / требует доработки |
| `[ ]` | Не сделано |
| `[x]` | Отменено |

В списках используется запись `- [v]`, `- [w]`, `- [ ]`, `- [x]` (пробел в незакрытом чекбоксе обязателен).

---

## Цель

Привести поведение пайплайна к правилам:
- загружать все строки из всех таблиц и стендов без исключений;
- сохранять дубли (включая дубли по ключу внутри одного файла);
- схлопывать строки по понятным правилам выбора и сравнения;
- корректно учитывать доп. колонки при сравнении;
- собирать подробные различия в `DIFF_REPORT`;
- формировать сводные отчеты консистентности (`CONSISTENCY`, `CONS_REPORT`).

## Детальный план доработок (1–20)

### Блок A. Базовая логика загрузки и merge

- [v] 1. Зафиксировать текущие расхождения между требованием и кодом
  - Проверены этапы `_parse_all_rows`, `_merge_entity_default`, `_merge_entity_with_optional_fields`.
  - Подтверждена запись в `raw_rows` и поведение при optional-полях.

- [v] 2. Исправить логику загрузки в SQLite (`raw_rows`)
  - Убрано ограничение записи только для `NEW`-файлов.
  - В `raw_rows` пишутся все прочитанные строки запуска с привязкой к `run_id`.

- [v] 3. Ввести единые правила "строки идентичны/разные" для схлопывания
  - Группировка по `business_key`.
  - Разделение/объединение выполняется по фактическому сравнению значений.

- [v] 4. Привести обработку доп. колонок к требуемому поведению
  - Учитываются пересечения/объединения колонок там, где это нужно.
  - Доп. поля корректно отражаются в итоговых структурах.

- [v] 5. Обновить формирование `source_stands`
  - Источники формируются по фактическому участию строк.
  - Стабилизирован порядок стендов.

- [v] 6. Уточнить выбор строки-эталона для вывода значений
  - Выбор эталона приведен к приоритетам.
  - Сохраняется предсказуемость вывода.

- [v] 7. Добавить тесты на критичные сценарии в `src/Tests`
  - Проверены сценарии дублей, ключей и расхождений.
  - Добавлены проверки приоритета выбора строки и служебных полей.

- [w] 8. Расширить логирование и диагностику
  - Добавлены части метрик и диагностики.
  - Нужно дорасширить DEBUG-детализацию по этапам.

- [w] 9. Валидация результата на полном наборе CSV
  - Базовые прогоны выполнены.
  - Нужна дополнительная формализованная верификация контрольных кейсов.

### Блок B. Консистентность и SPOD-совместимость

- [v] 10. Расширить движок `consistency_checks` до SPOD-совместимого формата правил
  - Поддержаны `sheet/sheet_src/sheet_ref`, `columns_src/columns_ref`, `json_key`, `column_compare`.
  - Нормализованы форматы и alias-имена листов.

- [v] 11. Доработать типы проверок по эталонному проекту
  - `unique`, `referential`, `referential_composite`, `json_field_equals_column`, `json_field_in_column`, `cross_sheet_date_lte_today`.

- [v] 12. Обновить `config.json` до расширенного набора правил
  - Добавлены и синхронизированы блоки правил и секции `csv_columns_count`.

- [v] 13. Реализовать новый формат листа `CONSISTENCY`
  - Свод по правилам и стендам.
  - Добавлены `check_id`, `violations`, `sample`.

- [v] 14. Добавить консольную таблицу нарушений после этапа консистентности
  - Строки: правила, колонки: `IFT`, `PROM`, `PSI`, `NO_STAND`, `TOTAL`.

- [v] 15. Привести входные файлы и `config.json` к единым именам стендов
  - Переименованы CSV и синхронизированы `file_names`.

- [v] 16. Применить к листу `CONSISTENCY` те же правила форматирования Excel, что и к остальным листам
  - Подключено общее форматирование и автоширина с лимитом.
  - Добавлен отдельный блок `excel.consistency_sheet`.

### Блок C. Дубли, отчеты различий и аналитика

- [v] 17. Перевести merge на выбор одной строки по `business_key` с приоритетом стендов
  - Приоритет: `PROM` -> `PSI` -> `IFT`.
  - Внутри выбранного стенда берется первая строка.

- [v] 18. Обновить служебные колонки по ключу
  - Добавлены `same_row_stands` и `same_key_diff_stands`.
  - Детали отличий убраны с листов сущностей.

- [v] 19. Расширить `DIFF_REPORT`
  - Добавлена подробная детализация по колонкам/позициям/сниппетам.
  - Добавлен CSV-экспорт `DIFF_REPORT` с управлением через `excel.diff_report_sheet.export_csv` (по умолчанию выключен).
  - В отчет попадают только реальные различия (без полностью совпадающих пар).

- [v] 20. Добавить агрегированный `CONS_REPORT`
  - Свод нарушений по сущностям/правилам/стендам.
  - Добавлена диагностическая детализация (`diff_columns`, `diff_positions`, `diff_snippets`) в формате, близком к `DIFF_REPORT`.
  - Добавлен CSV-экспорт `CONS_REPORT` с управлением через `excel.cons_report_sheet.export_csv` (по умолчанию выключен).
