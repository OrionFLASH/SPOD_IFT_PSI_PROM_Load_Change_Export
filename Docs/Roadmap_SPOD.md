# Roadmap: консолидация настроек SPOD

Документ отражает **плановые этапы**, **фактическую реализацию** и **бэклог**. Условные обозначения:

| Обозначение | Значение |
|-------------|----------|
| **[сделано]** | Реализовано в коде и используется по умолчанию |
| **[частично]** | Есть задел или ручная проверка, нет полной автоматизации в коде/CI |
| **[не сделано]** | Не реализовано или не внедрено в репозиторий |

---

## Общий статус (сводка)

| Область | Статус |
|---------|--------|
| ETL (сканирование, парсинг, merge, Excel, SQLite) | **[сделано]** |
| Конфигурация `config.json` (пути, сущности, форматирование, runtime) | **[сделано]** |
| Pre-check обязательных файлов | **[сделано]** |
| Параллельная обработка и `dry-run` | **[сделано]** |
| Лист `DIFF_REPORT` | **[сделано]** |
| Unit-тесты в репозитории | **[частично]** (3 теста в `src/Tests/test_pipeline.py`) |
| Автоматический CI (GitHub Actions и т.п.) | **[не сделано]** |
| Формальные миграции схемы БД | **[не сделано]** |
| Валидация схемы CSV по обязательным колонкам | **[не сделано]** |

### Этап 1. Каркас проекта

**Статус: [сделано]**

- [сделано] Модули: `config`, `logging_setup`, `pipeline`, `models`, `main`.
- [сделано] Структура каталогов `src`, `src/Tests`, `IN`, `OUT`, `log`, `Docs`.

---

### Этап 2. Инжест

**Статус: [сделано]**

- [сделано] Чтение CSV из трёх стендов (`IFT`, `PROM`, `PSI`), расчёт `SHA-256`, регистрация в SQLite.
- [сделано] Имена файлов задаются в `config.json` отдельно по стенду и сущности.
- [сделано] Pre-check: отсутствие обязательных файлов/каталогов блокирует запуск с перечислением пропусков.
- [сделано] Пакетная запись в БД (`executemany`) для производительности.

---

### Этап 3. Нормализация и сравнение

**Статус: [сделано]**

- [сделано] Нормализация строк, `row_hash`, `business_key`, fallback-ключ `HASH:*`.
- [сделано] Флаги `same_key_diff_values_flag`, `same_key_diff_values_note`.
- [сделано] Optional-поля для выбранных сущностей (например `REPORT`, `EMPLOYEE`): эталон `reference_stand`, сравнение только если поле есть в обеих строках.
- [сделано] Служебные поля diff в выводе и конфиге: `diff_columns`, `diff_positions`, `diff_snippets` (часть в `service_fields` / настройках листа).

---

### Этап 4. Формирование merged-набора

**Статус: [сделано]**

- [сделано] `source_stands`, `source_count`, `is_equal_all`, `diff_group_key`.
- [сделано] Разделение строк при коллизии `row_hash` при разном «сыром» наборе полей; эталон значений для отображения — `merge.reference_row_stand` (по умолчанию `PROM`).
- [сделано] Расширенная статистика разночтений в консоли и логах.

---

### Этап 5. Экспорт Excel

**Статус: [сделано]**

- [сделано] Лист `SUMMARY` и по одному листу на сущность (10 типов).
- [сделано] Закрепление областей, автофильтр, автоподбор ширины с лимитом, перенос текста, выравнивание.
- [сделано] Рамки данных (пунктир/точки), двойной разделитель под заголовком, толстая вертикальная граница CSV / служебного блока.
- [сделано] Подсветка строк (в т.ч. отсутствие в `PROM`, конфликт значений) с приоритетом из конфига.
- [сделано] Подсветка заголовков ключевых полей и доп.полей (зелёный для доп.).
- [сделано] Выход в `OUT/XLS`, имя файла: `output_name_prefix` + `output_timestamp_format`.
- [сделано] Лист **`DIFF_REPORT`** (включается через `excel.diff_report_sheet.enabled` в `config.json`) — агрегированная сводка конфликтов.

### Этап 8. Тестирование

**Статус: [частично]**

- [сделано] Интеграционные и регрессионные прогоны на данных (сценарии S0–S4), результаты в `Docs/TestReports/` (в т.ч. `test_results_regression_latest.md`).
- [не сделано] Автозапуск интеграционных сценариев в CI при каждом коммите.
- [не сделано] Unit-тесты на правила подсветки Excel и все ветки форматирования из конфига.
- [не сделано] Интеграционные тесты на «больших» наборах CSV внутри репозитория (без внешних данных).
