# Roadmap: Консолидация настроек SPOD

## Общий статус

- Статус проекта: **этап базовой реализации завершен**.
- Основные модули ETL, конфиг, логирование, SQLite и Excel-экспорт уже реализованы.

## Этапы и статус выполнения

1. Этап 0. Формализация ТЗ  
   Статус: **выполнено**
   - Зафиксированы сущности, ключи, правила эквивалентности.

2. Этап 1. Каркас проекта  
   Статус: **выполнено**
   - Созданы модули: `config`, `logging`, `pipeline`, `models`, `main`.

3. Этап 2. Инжест  
   Статус: **выполнено**
   - Реализовано чтение CSV из 3 стендов, расчет hash, регистрация файлов в SQLite.

4. Этап 3. Нормализация и сравнение  
   Статус: **выполнено**
   - Реализована нормализация строк, `row_hash`, `business_key`.

5. Этап 4. Формирование merged-набора  
   Статус: **выполнено**
   - Реализованы `source_stands`, `source_count`, `is_equal_all`, `diff_group_key`.

6. Этап 5. Экспорт Excel  
   Статус: **выполнено**
   - Реализован Excel с листом `SUMMARY` и листами по сущностям.

7. Этап 6. Логирование и отчеты  
   Статус: **выполнено (базово)**
   - Настроены INFO/DEBUG логи по заданному формату.
   - Требуется расширение аналитических отчетов при необходимости.

8. Этап 7. Тестирование  
   Статус: **выполнено (базово)**
   - Добавлены unit-тесты для ключевых правил формирования ключей.
   - Требуется расширение интеграционных тестов для edge-case сценариев.

9. Этап 8. Документация  
   Статус: **выполнено**
   - Добавлены `README.md`, `config.json`, документация по требованиям и roadmap.

## Следующие шаги (рекомендуемые)

- Добавить интеграционные тесты на больших наборах CSV.
- Добавить валидацию схем входных файлов по каждой сущности.
- Добавить расширенный отчет по расхождениям между стендами (отдельный лист в Excel/JSON-отчет).
- Опционально добавить CLI-флаг `--dry-run` с детальным результатом без записи в БД.
