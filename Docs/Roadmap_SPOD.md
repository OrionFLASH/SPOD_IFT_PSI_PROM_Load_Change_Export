# Roadmap: Консолидация настроек SPOD

## Общий статус

- Статус проекта: **этап расширенной реализации завершен**.
- Реализованы: ETL, конфигурируемые имена файлов по стендам, консольная аналитика процесса, SQLite-дедупликация, расширенный Excel-экспорт и форматирование листов через `config.json`.

## Этапы и статус выполнения

1. Этап 0. Формализация ТЗ  
   Статус: **выполнено**
   - Зафиксированы сущности, ключи, правила эквивалентности.

2. Этап 1. Каркас проекта  
   Статус: **выполнено**
   - Созданы модули: `config`, `logging`, `pipeline`, `models`, `main`.

3. Этап 2. Инжест  
   Статус: **выполнено**
   - Реализовано чтение CSV из 3 стендов, расчет hash, регистрация файлов в SQLite.
   - Реализована конфигурация явных имен файлов по стендам в `config.json` (поддержка разных имен).
   - Добавлена жесткая pre-check в начале: отсутствие обязательных файлов/каталогов блокирует запуск с подробным списком пропусков.

4. Этап 3. Нормализация и сравнение  
   Статус: **выполнено**
   - Реализована нормализация строк, `row_hash`, `business_key`.
   - Добавлены флаги `same_key_diff_values_flag` и `same_key_diff_values_note` для ключей с разным содержимым.
   - Добавлена поддержка доп.полей относительно `PROM` для `REPORT` и `EMPLOYEE`:
     - отсутствие доп.поля не считается отличием;
     - значение доп.поля сравнивается, если поле есть в обеих сравниваемых строках.

5. Этап 4. Формирование merged-набора  
   Статус: **выполнено**
   - Реализованы `source_stands`, `source_count`, `is_equal_all`, `diff_group_key`.
   - Добавлена расширенная статистика разночтений по сущностям и в целом по запуску.

6. Этап 5. Экспорт Excel  
   Статус: **выполнено**
   - Реализован Excel с листом `SUMMARY` и листами по сущностям.
   - Реализованы: закрепление областей, автофильтр 1-й строки, автоподбор ширины с лимитом, перенос текста, стили заголовка/данных.
   - Добавлены: рамки данных (пунктир/точки), двойной разделитель заголовка, толстый вертикальный разделитель служебного блока.
   - Реализована подсветка строк:
     - светло-желтым — если строка отсутствует в `PROM`, но есть в `IFT` и/или `PSI`;
     - светло-красным — если один ключ имеет разные значения.
   - Добавлена подсветка заголовков ключевых полей светло-оранжевым.
   - Добавлена подсветка заголовков доп.полей светло-зеленым.
   - Реализован приоритет правил заливки строк через конфиг.
   - Все параметры форматирования вынесены в `config.json`.
   - Выходные файлы Excel перенесены в `OUT/XLS`.
   - Имя файла вынесено в составные настройки (`output_name_prefix` + `output_timestamp_format`).

7. Этап 6. Логирование и отчеты  
   Статус: **выполнено**
   - Настроены INFO/DEBUG логи по заданному формату.
   - Добавлен потоковый консольный отчет: старт, найденные файлы, прогресс обработки, статус обновления БД, количество совпадений и разночтений.

8. Этап 7. Тестирование  
   Статус: **выполнено (базово)**
   - Добавлены unit-тесты для ключевых правил формирования ключей.
   - Требуется расширение интеграционных тестов для edge-case сценариев.

9. Этап 8. Документация  
   Статус: **выполнено**
   - Добавлены `README.md`, `config.json`, документация по требованиям и roadmap.
   - Документация актуализирована под новую схему конфигурации и форматирование Excel.

## Следующие шаги (рекомендуемые)

- Добавить интеграционные тесты на больших наборах CSV.
- Добавить валидацию схем входных файлов по каждой сущности.
- Добавить отдельный лист `DIFF_REPORT` со сводкой конфликтов по ключам и полям.
- Добавить CLI-флаг `--dry-run` с детальным результатом без записи в БД.
